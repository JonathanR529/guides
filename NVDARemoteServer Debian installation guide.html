NVDARemoteServer Debian installation guide

In this guide, you will learn how to set up and install the NVDARemote relay server on Debian based systems.

Step 1: Install git

You will need git to clone the repository from GitHub. Clone is another term for downloading. To install git, execute the following command:

sudo apt install git -y

The reason we include the sudo flag at the beginning of the commands is because the apt command makes changes to the system at the root level.

Step 2: cd to a directory to where you want to store the contents of the server

The server will be removed, so just pick any directory of your choosing. Example: cd /home/user

git clone https://github.com/jmdaweb/NVDARemoteServer.git

The reason we include the .git at the end of the link, is because it tells the git program the specific repository we are trying to download.

Step 3: cd to the NVDARemoteServer/debian directory:

Once git finishes cloning the repository, we are going to cd to the NVDARemoteServer/debian directory to execute commands to the NVDARemoteServer build script for the Debian operating system:

cd NVDARemoteServer/debian

Step 4: Change file permissions and execute the build script

The reason we want to change file permissions is that by default, the file does not have permissions to write changes to the system.

chmod +x build.sh

Step 5: Execute the build script and build the package

Once file permissions are changed, run the following two commands to execute the build script and build the package:

sudo ./build.sh
sudo dpkg -i nvda-remote-server_2.3.deb

At the time of this writing, the version number is 2.3, but it always changes. Check the nvda-remote-server_version file in the same directory to make sure.

ls

Step 6: Starting the NVDARemoteServer at system startup

We want the NVDARemoteServer to start at system startup because we don't want to start it each time the system reboots.

sudo systemctl enable NVDARemoteServer

Now, use the following command to start the NVDARemoteServer:

sudo systemctl start NVDARemoteServer

Final notes

Go back to the previous directory, and then you can now remove the NVDARemoteServer directory.

Use the following commands to go back to the previous directory, and remove the NVDARemoteServer directory:

cd ..
sudo rm -rf NVDARemoteServer
